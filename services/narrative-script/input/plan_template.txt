# 動画企画書テンプレート：4段階ナラティブ構成

## 1. 企画の核心（Core Concept）

この構成が目指すのは、視聴者の「解釈の更新」だ。

「あの出来事はそういう構造だったのか」という、
静かな認識のアップデートを持って帰ること——それがゴールである。

そのためにすべきことは、感情を直接語ることではない。
主人公が直面した事実・出来事・エピソードを積み重ね、
視聴者が自分で「なぜそうなったのか」「どうすれば良かったのか」を問い始めるよう設計する。

Chronicle of Discoveryで積み上げた未解決の課題が、
Schema Updateで主人公の行動によって解決されるカタルシスを通じて、
視聴者は自分がこれまで持っていた見方を静かに書き換える。

ただしこの体験は、動画を「最後まで見た者」にしか届かない。
だから最初にすべきことは、解釈の設計ではなく、
視聴者が離脱できない状態を作ることだ。

---

## 2. 【Set Up】強力なフック

**目的**：視聴者の「常識・予測・信念」を冒頭で裏切り、
脳が「答えを得るまで離脱できない」状態を作る。

感情移入・共感はここでは不要。
このセクションの唯一の機能は「続きを見ざるを得ない認知状態の生成」。
感情ではなく、認知の仕組みに働きかける。

人間の脳は「予測の裏切り」を自動的に処理しようとする。
予測が外れた瞬間、脳はその不一致を解消するまで
注意リソースをその対象に集中させ続ける。
Set Upはこの仕組みを利用する。

**設計の4原則**：

1. **断片的事実によるフック（説明の完全排除）**
   「〇〇とは〜である」「1977年、NASAは〜」といった背景説明・導入・定義は一行たりとも書かない。
   カメラが捉えた「異常な一場面」や「理解不能なディテール」の描写から始め、
   視聴者の脳内に巨大な情報の空白（知識の欠如）を強制的に作り出す。

2. **信念の裏切り**
   視聴者が「当然そうだ」と思っている常識・結末・人物像を、
   冒頭の数十秒以内に静かに、しかし確実に裏切る。
   「えっ、そうじゃないのか」という認知的不協和が発生した瞬間、
   視聴者は自発的に「なぜ？」を問い始める。
   驚かせるのではなく、「自分の予測が間違っていた」と気づかせること。

3. **情報の非対称性の設計**
   視聴者が「何かを知らされていない」と感じる状態を作る。
   語りすぎず、しかし無視できない何かを提示する。
   「この先に、自分がまだ知らない重要な事実がある」という確信を
   視聴者自身に形成させること。

4. **離脱コストの刻印**
   「ここで動画を閉じたら、何かを永遠に知らないまま終わる」という感覚を
   暗示的に植え付ける。
   明示的な「続きを見てください」は禁止。
   視聴者が自分の意志で「続けなければ」と感じる構造にする。

**素材として用意すべきもの**：
- 視聴者の常識を裏切る「逆説的な事実」または「予想外の結末の断片」
- 「なぜそうなったのか」を知らずには終われない「認知の引っかかり」
- 動画全体の問いへの「予感」（問い自体はまだ提示しない）
- ※ネタバレ（結末・核心的な展開の先出し）は厳禁

**テンプレートを埋める際の注意**：
- 素材として書くのは「何が起きたか（事実）」のみ。意味・感情・解釈は書かない。
  「その状況は〇〇だった」「これはつまり〇〇という苦しさだ」といった解釈はAIに渡さない。
  代わりに、その状況を引き起こした具体的な事実やエピソードを1文追加して代替する。

## 3. 【Dramatic Question】核心の問いの設計素材

**目的**：舞台の報酬・損失構造を暗示的に確立し、
視聴者が「具体的な結末を予測せずにいられない」問いを刻む。

このパートの機能は2段階。
①まず「何がかかっているか」を視聴者に理解させる（報酬/損失の構造設計）
②その上で、答えが見えそうで見えない問いを暗示的に提示する。
②の問いは、視聴者が自分なりの予測を立てられる粒度でなければならない。
「どうなるんだろう」ではなく「きっとこうなる、でも本当にそうなのか」まで
視聴者を連れて行くこと。

**ステップ1：舞台と賭け金の設計**

問いを立てる前に、視聴者が「この物語で何が報酬で、何が損失か」を
直感的に理解している必要がある。

報酬と損失は明示しない。
主人公の置かれた状況、関係性、これまでに積み上げてきたもの、
失った場合に二度と取り戻せないもの——
これらを描写することで、視聴者が自分で「賭け金」を算出する。

視聴者が自分で計算した賭け金は、外から与えられたものより重い。

設計すべき要素：
- **報酬の輪郭**：主人公が「望む結末」に辿り着いた場合、何が手に入るか
  （成功・承認・救済・真実・関係の回復、など）
- **損失の輪郭**：失敗した場合に失われるものが、
  「取り返しのつかない喪失」であることを暗示する
  （死・崩壊・永続的な後悔、など——ただし直接的に語ってはならない）
- **非対称性**：報酬より損失の方が重く感じられるよう設計する。
  人間は損失回避の方が動機として強く働く。

**ステップ2：問いの刻印**

賭け金が視聴者の中に形成された後、問いを提示する。

問いは「達成できるか」に限らず、
「守れるか」「逃げ切れるか」「信念を貫けるか」「真実にたどり着けるか」など、
物語の核心に応じて設定する。

ただし問いは**暗示的に**示す。
「この動画では〇〇を問います」と宣言するのではなく、
物語の文脈の中に問いを溶け込ませ、視聴者が「自分で問いに気づいた」と感じる形にする。

**問いの粒度（重要）**：
問いは「どうなるんだろう」という抽象的な疑問では不十分。
「AになるのかBになるのか」と視聴者が具体的な結末を想像できる粒度まで絞り込む。
例：「地球の撮影に成功するか」「この沈黙の先に、救いはあるのか」「かつての仲間を、彼は撃てるのか」

問いが正しく機能しているかの判定基準：
- 視聴者が「たぶん〇〇になると思う、でも……」と具体的な予測を立てられるか
- その予測が外れるかもしれないという不安が同時に存在するか
- 答えを知らずに動画を閉じることへの抵抗感が生まれているか

この3つが揃ったとき、問いは「心理的拘束」として機能している。

## 4. 【Chronicle of Discovery-1】課題・葛藤の蓄積素材

**目的**：主人公が直面した事実・出来事・エピソードを積み重ね、未解決の課題と葛藤を蓄積させる。感情の名前は語らない。

探究の軌跡は、主人公が問いへの答えを求めてもがく過程であり、最終パート（Schema）で解決されることになる課題を、事実とエピソードとして視聴者に積み上げていく工程である。
「小さな前進 → それを阻む新たな課題・壁 → それでも引き受けて進む」というwave構造を繰り返し、未解決の課題の積み重なりを深めていく。

感情を直接語るのではなく、主人公が置かれた状況・行動・出来事を具体的に描写することで、視聴者が自分でその重さを感じ取れる余白を残す。

- **課題・葛藤の設計**：
  どのような事実・出来事・矛盾が積み重なるか。感情ではなく「起きたこと」として描写する素材を用意する。
- **情景描写の素材**：
  各エピソードは「場所・時間・人物の動作」という冒頭1文で立ち上げてから書く。
  視覚・聴覚・触覚・嗅覚など五感を刺激するディテール（錆びた鉄の匂い、深夜に響く足音、モニターの青白い光など）の素材を用意する。カメラが捉えられる事実のみを書くこと。
- **拘束の描写**：
  主人公がなぜ立ち止まれないのか、なぜ引き受けざるを得ないのかという「状況の拘束」を、具体的な行動・数字・出来事として描く。
- **次のパートへのフック**：
  各エピソードの末尾に、「まだ説明されていない矛盾・課題」を一事実として置く。視聴者が「なぜそうなったのか？」と問い続けたくなる具体的な引っかかりを設計する。
- **Schema Updateとの関係**：
  この課題の蓄積が、最終パートでのカタルシス（解釈の更新）の深さを決めることを意識する。

## 5. 【Chronicle of Discovery-2】（必要に応じて追加）

**目的**：課題の積み重なりをさらに深め、最終パートでの解決への必然性を高める。

- **詳細**：
  中盤の展開。Schemaパートで解決されることになる課題（構造的な矛盾、引き受けざるを得ない状況、未解決の問いなど）が、物語の文脈の中で事実とエピソードとして積み重なっていく描写。
  感情の名前は語らず、「起きた出来事」「取った行動」「その結果」を具体的に描写する。

- **課題蓄積のwave構造**：
  小さな前進と、それを阻む新たな課題・矛盾。それでも引き受けて続けた事実をどう描くか。

- **このパート末尾のフック**：
  Schema Updateへ向かう必然性を感じさせる、未解決の課題または「まだ説明されていない行動・状況」を一事実として置く。

## 6. 【Schema Update】価値観・解釈の更新とカタルシスの設計素材

**目的**：Chronicle of Discoveryで蓄積された課題が主人公の行動によって解決されるカタルシスを通じて、視聴者の価値観・解釈を静かに更新する体験を設計する。

重要なのは**「課題の解決場面」と「解釈の文脈化」**だ。感情や価値観は直接語らない。すべては事実とエピソードによって示す。

主人公が何かを決断する・手放す・あるいはそれでも続ける場面を、具体的な行動・出来事として描写する。
「解決した」「わかった」と説明するのではなく、解決が起きた場面を目撃させることで示す。
視聴者の中で「あの出来事はそういう構造だったのか」という認識の書き換えが、自然に起きる構造を目指す。

Chronicle of Discoveryで積み上げた課題を「実は美しかった」と逆転するのではなく、「そういう構造になっていた」と示す事実・エピソードを置く。
価値判断は下さない。ただ「そういう事実があった」と描くことで、視聴者は自分で解釈を更新できる。

解決は、視聴者の予測を裏切りつつも、後から考えれば「それしかなかった」と感じさせるものでなければならない。
カタルシスは、主人公が報われたかどうかではなく、
「課題が解決される瞬間の目撃」によって、視聴者自身の解釈が静かに書き換わることから生まれる。

**逆転のルール**：
解決は必ず伏線の回収として機能させる。
唐突な奇跡・外部からの救済となる素材は使わないこと。
主人公は必ず何かを失う形で解決を描く。

**文体の緩急（カタルシス設計の核心）**：
カタルシスの手前では、解決への圧縮感・焦燥感を短文の連打で表現する素材を用意する。
（例：「彼は手を止めなかった。」「カメラが動いた。」「針が振れた。」のような、
 誰が・何をしたかが1文で伝わる短い事実の連鎖。体言止め・意味の通じない名詞の羅列は禁止）
カタルシスの瞬間以降は、静けさと余韻をゆったりとした長文で描く素材を用意する。
この「切り替え点」（圧縮→解放の転換点）がどの瞬間かを素材に明記しておく。

**余韻の設計**：
主人公のその後を短く、静かに描写する素材を用意する。
最後の場面は「解釈を言葉で説明する」のではなく、「情景として示す」。
「見方が変わった」「孤独ではなかった」などの解釈の言語化は禁止。
ただ「そういう事実があった」という情景を置き、視聴者が自分で解釈を更新する余白を残す。

## 7. 作品のトーン・キャラクター指示

- **全体の雰囲気**：（静寂、抑制、余白重視、など。感情を煽るより感情の器を作る方向で）
- **解説役のトーン**：目撃者としての語り。断定せず、ただ描写する。
  「だ・である」調だが書き言葉の硬さを避け、口語寄りの自然な文体にする。
  相槌・応答は性別を感じさせない表現を使うこと（「ああ」「いや」などの男性的な口調は禁止。「うん」「まさに」「その通りだ」などを使う）。
- **視聴者の反応イメージ**：
  「救われた気持ちになる」「自分のことを言われた気がした」「泣いた理由がよくわからない」
